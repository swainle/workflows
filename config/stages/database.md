- DBML 表名以 `t_` 开头。
- 时间字段使用 `create_at` 和 `update_at`，由数据库维护。
- 需要表达数据表关系时，在 Markdown 中使用 Mermaid `erDiagram`。
- 根据实际过滤、排序、关联和唯一性规则设计单列、联合、唯一或部分索引，说明字段顺序和对应查询场景。
- 需要时使用 `EXPLAIN` 或 `EXPLAIN ANALYZE` 验证索引，并评估写入和存储成本，不为每个字段机械添加索引。
- 阶段产物必须包含约束、迁移、历史数据、回滚和关键查询性能的验证方案。
- 数据库和 ORM 未明确时写入 `# 待确认问题`，不自行指定框架。
