# 本阶段要做什么

说明当前需求新增或改变的表、字段、关系、索引、约束和历史数据处理。

# 技术选型确认

- 生成 DBML 前必须先向使用者逐条确认数据库产品及主版本，再确认 ORM 框架及主版本；允许明确选择“不使用 ORM”。
- 每次只确认一个选型。可以先从现有配置和源码识别候选项并说明依据，但不得仅根据默认文件自行决定；已有本次人工确认记录时不重复询问。
- 两项都得到明确确认后才能生成文件。DBML 的 `database_type`、字段类型、默认值、索引和约束必须适配所选数据库；模型映射与迁移说明必须适配所选 ORM。
- 在分析文件中记录最终确认的数据库、ORM、版本和依据，避免后续阶段重新猜测。

# 表与时间字段规范

- AI 新增的表名必须以 `t_` 开头。
- 每张新表必须包含 `create_at` 和 `update_at`，不得改成 `created_at` 或 `updated_at`。
- `create_at` 由数据库在插入时自动写入当前时间，应用代码不得赋值。
- `update_at` 由数据库在插入时写入、在每次更新时自动刷新，使用数据库原生能力或触发器实现，应用代码不得赋值。
