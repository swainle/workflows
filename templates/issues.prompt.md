# 本阶段要做什么

分析全部 GitHub Issues，为主 Issue 创建或完善 `01-prd.md`。

- 需求目录编号必须与主 Issue 对应，例如 Issue `#36` 对应 `REQ-036-*`。
- 找出重复、合并、保留、排除和待确认的 Issues。
- 待确认的 Issue 本次不修改、不合并进当前 PRD，留到下次需求流程。
- 每个相关 Issue 都要在分析 Markdown 的简单表格中写明处理方式和原因。
- 已有 `01-prd.md` 可能被人工修改，必须在它的基础上优化。

# 允许修改

- `{{REQUIREMENT_DIR}}/01-prd.md`
- `{{REQUIREMENT_DIR}}/01-*.md`

不得创建、修改或关闭 GitHub Issue，不得修改业务源码和全局文件。
